В общем, я решил написать своё ТЗ, в качестве основы в котором я переиначил, оптимизирован и добавил пару задач, которые лично мне приходилось реализовывать на реальных проектах

Предлагаю сделать систему (что-то около CRM в первом приближении) по управлению складами и взаимодействию со складами партнеров. Ниже опишу основной бизнес-процесс:

На вашей стороне есть n число складов, каждый из которых хранит в себе различные товары по номенклатуре (определенный список товаров) в определенном кол-ве.
Каждый из товаров имеет свои характеристики, а также условную единицу грузоместа (сколько одна единица товара занимает в транспорте)
Заполнение новых партий товаров на конкретном складе выполняется через API, грубо говоря склад, товар и сколько единиц станет
У вас также есть информация о складах партнеров - расстояние до других складов (или, что лучше, координаты). И ваши склады, и склады партнеров имеют возможные транспорты для доставки (авто, ж/д, авиа). Склад может любую комбинацию способов доставки, но как минимум один тип должен быть всегда. У транспорта также есть вместимость, скорость и стоимость за км движения
Партнер склада вызывает API - в него он передает вид товара, количество, свой склад и желаемое условие доставки (быстрее или дешевле). В результате ответа должен отработать алгоритм, который рассчитает список самых релевантных вариантов доставки с учетом запроса, а также описанных выше условий хранения и доставки в системе.

В целом, я предполагаю, что это будет чисто бек система (без UI), которая будет работать через Postman. Если у вас есть желание - не запрещаю прикрутить JS, или если на самой Java - Vaadin или какой-либо из шаблонизаторов. Также есть варианты по определению расстояния между складами - или хранить просто в БД через связь многие-ко-многим (простой вариант), или поработать через координаты (написать алгоритм свой или произвести интеграцию с библиотеками/публичными API в Интернете).
